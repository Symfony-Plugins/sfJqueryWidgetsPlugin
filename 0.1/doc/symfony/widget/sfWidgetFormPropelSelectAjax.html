Available options:

  • url: The url of the action(routing, or module/action) to be called using Ajax (required)
  • update: The ID of the HTML element to be updated (model_name_update by default)
  • with: List of parameters to be sent with the Ajax request (value of the selected select option is added by default as a parameter called id)
  • complete: A function to be called when the request finishes (after success and error callbacks are executed, null by default)
  • error: A function to be called if the request fails (null by default)
  • beforeSend: A pre-callback to modify the XMLHttpRequest object before it is sent
  • event: The DOM event of the select tag that will trigger the Ajax call (onchange by default)
  • update_element: The HTML element rendered that will be updated with the Ajax response (div by default)
Usage:

The following example creates a drop down list that sends an Ajax request to the action "test/ajax" on change of the select menu. By default if you don't specify the update option, the widget renders an empty div element with an id in the format model_name_update.

  1.     $this->widgetSchema['category'] = new sfWidgetFormPropelSelectAjax(array(
  2.         'model' => 'Category',
  3.         'method' => 'getName',
  4.         'url' => 'test/ajax'
  5.     ));

The above code will render the following HTML code

  1.     <label for="category_category">Category</label>
  2.     <select id="category_category" name="category[category]">
  3.       <option value="2">Fun</option>
  4.       <option value="3">Stuff</option>
  5.     </select>
  6.     <div id="category_update"><div>
  7.     <script charset="utf-8" type="text/javascript">
  8.       jQuery('#category_category').change(function(event){
  9.         jQuery.ajax({
  10.           type:        'POST',
  11.           dataType:    'html',
  12.           url:         '/sandbox/web/backend_dev.php/test/ajax',
  13.           data:        "id="+jQuery('#'+event.target.id).val(),
  14.           success:
  15.             function(data, textStatu)
  16.             {
  17.               jQuery('#category_update').html(data);
  18.             }
  19.  
  20.         })
  21.       });
  22.     </script>

If you want to change the id of the generated div or change the div to lets say a span use the following

  1.     'update' => 'foobar'

  1.     'update_element' => 'span'

NOTE: If you add the update option with a value other than the default one added automatically by the widget you will have to add the HTML element to be updated manually in the template.
